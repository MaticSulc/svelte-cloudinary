## Using Environment Variables

If you're using Vite/SvelteKit the following environment variables are supported:

- `VITE_CLOUDINARY_CLOUD_NAME`
- `VITE_CLOUDINARY_API_KEY`
- `VITE_CLOUDINARY_PRIVATE_CDN`
- `VITE_CLOUDINARY_SECURE_DISTRIBUTION`
- `VITE_CLOUDINARY_UPLOAD_PRESET`

## Config Helper

If you're unable to use [environment variables](#using-environment-variables), or there isn't one for a config option you want to set *globally*, you can use the `configureCloudinary` function. It uses the Svelte [`context api`](https://learn.svelte.dev/tutorial/context-api), so it needs to be called in your root Svelte component (or root layout in SvelteKit). Since context flows down in Svelte, any children of this root component will automatically be able to detect the Svelte Cloudinary config.

```svelte
<script>
	import { configureCloudinary } from 'svelte-cloudinary';

	configureCloudinary({
		cloudName: 'cloudName',
	});
</script>
```

The following options are available:

```ts
// These options are available on every helper/component
// (see Config Overrides below)
interface ConfigOptions {
	cloud?: {}
	url?: {}
}

interface GlobalCloudinaryConfig extends ConfigOptions {
	analytics?: AnalyticsOptions;
	uploadPreset?: string;
	cloudName?: string;
	apiKey?: string;
}
```

## Config Overrides

You can overwrite the config on individual components/helpers. All components have a `config` prop which accepts the Cloudinary `ConfigOptions` object. This doesn't include the shortcuts such as the top level `cloudName`, and `apiKey` that the global config does.

```svelte
<script>
	import { CldImage } from 'svelte-cloudinary';
</script>

<CldImage config={{ cloud: { cloudName: 'cloudName' }}} />
```

## Priority

Config options are used in the following order:

- [Options overrides](#config-overrides)
- [`configureCloudinary`](#config-helper): `.cloudName` and `.apiKey`
- [`configureCloudinary`](#config-helper): `.cloud` and `.url`
- [Environment variables](#using-environment-variables)
